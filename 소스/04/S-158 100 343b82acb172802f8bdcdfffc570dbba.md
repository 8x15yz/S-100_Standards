# S-158:100

# S-158:100 공통(Generic) 검증 규칙

---

S-158:100은 S-100 기반 데이터 제품을 검증하는 소프트웨어가 반드시 구현해야 하는 공통(Generic) 검증 항목을 정의한 규칙입니다. 본 지침에서는 S-158:100을 모든 S-100 제품에 공통 적용되는 유효성 검증 규칙으로 설명하고 있으며, S-100 Edition 5.2.0 이상을 기반으로 작성된 모든 제품표준의 데이터셋 및 교환셋 전체에 적용되는 검증 체계로 제시하고 있습니다.

## 표준 개요

---

S-158:100은 S-100 기반 데이터 제품의 유효성 검증을 위한 공통 규칙 세트로, 제품명세(Product Specification)별 고유 규칙이 아닌 모든 S-100 제품에 공통적으로 적용되는 검증 기준을 제공합니다. 문서에서는 Geometry, Attribute, Metadata 등 서로 다른 제품에서 반복적으로 사용되는 개념들에 대해 일관된 검증 품질을 확보하는 역할을 수행한다고 설명하고 있습니다.

또한 S-158:100은 단독으로 전체 검증을 완료하는 규격이 아니라, 반드시 제품별 검증 규칙인 **S-158:1xx** 및 제품 간 상호운용성 검증 규칙인 **S-158:98**과 함께 사용되어야 하는 공통 기반 규칙으로 제시됩니다. 즉, S-158:100은 전체 S-158 검증 체계 중에서 공통 규칙 층을 담당하는 문서입니다.

## 적용목적 및 대상

---

S-158:100의 목적은 S-100 기반 데이터 제품을 검증하는 소프트웨어가 반드시 구현해야 하는 공통(Generic) 검증 항목을 정의하는 데 있습니다. 문서에는 다음과 같은 적용 방향이 제시되어 있습니다.

- 다수의 S-100 제품(S-101, S-102, S-104, S-111 등)이 공통적으로 준수해야 하는 검증 항목 정의
- 서로 다른 제품에서 동일 개념(Geometry, Attribute, Metadata 등)을 사용할 때 일관된 검증 품질 확보
- 데이터셋 및 교환셋 전체를 공통 구조 아래에서 검증

적용 대상은 다음과 같습니다.

- S-100 Edition 5.2.0 이상을 기반으로 작성된 모든 제품표준
- 데이터셋 및 교환셋 전체
- S-158 공통 구조(체크 번호, 문법, 분류 등)를 준수해야 하는 검증 항목

## 모델 구성

---

S-158:100은 제품 데이터 모델을 설명하는 규격이 아니라, 검증 항목의 구조와 검증 순서, 그리고 공통 검증 범주를 정의하는 규칙 체계입니다. 따라서 기존 S-1xx 제품표준의 데이터 모델 구성과는 다른 형식으로 이해하는 것이 적절합니다.

### 1. 체크 구조(Check Structure)

S-158:100의 각 검증 항목은 S-158에서 정의한 공통 구조를 그대로 사용하며, Validation Check는 다음 필드를 포함해야 합니다.

- **Check ID**: 고유 식별 번호
- **Classification**: Critical / Error / Warning 등 체크의 중대도
- **Check Message**: 오류 발생 시 사용자에게 전달되는 안내 문구
- **Check Description**: 검사 조건 및 위반 상황 기술
- **Check Solution**: 위반 상황에 대한 해결안 제시
- **Reference**: 참조하는 규격 및 문서 절(S-100, S-98, S-1xx, Part X 등)

### 2. 체크 번호 규칙

S-158:100의 체크 번호는 다음 형식을 사용합니다.

- 100_<4자리번호><선택적 소문자>

여기서 4자리 번호는 단순 고유 식별자이며, 소문자는 하나의 체크가 파생 체크로 분리될 때 부여됩니다.

### 3. 체크 종류(Collection A / B)

문서에서는 S-158:100의 체크를 두 가지 컬렉션으로 구분합니다.

- **Collection A**
    - 데이터셋·교환셋 검증용
    - 실제 검증 소프트웨어가 데이터셋 및 교환셋을 검사할 때 수행해야 하는 항목
- **Collection B**
    - 제품명세·FC·PC·GML Schema 작성 검증용

### 4. 단일 데이터셋 검증 순서

S-158:100에 특화된 내부 논리 순서는 다음과 같이 제시됩니다.

- **1a Spatial integrity**
    - 공간 무결성 검사
    - CRS, Geometry 유효성, 좌표 범위, Tolerance 검증
- **1b Thematic integrity**
    - 속성(주제) 무결성 검사
    - Attribute 값, datatype, enumeration, multiplicity 검증
- **1c Association integrity**
    - Feature/Information 관계 무결성 검사
    - Association 존재 여부, role, target 일치성 검증
- **1d Dataset structural conformance**
    - 데이터 구조 검사
    - Dataset ID, Metadata 구조, Part 10 규격 준수 여부 검증

## 주요 검증 항목 예시

---

문서 부록의 S-158:100 공통(Generic) 검증 항목 **– Collection A**에는 실제 검증 항목이 표 형식으로 제시되어 있습니다. 여기에는 Check ID, 구분, Check Message, Check Description, Check Solution, 참조 문서 및 조항이 포함되어 있으며, 공통 검증 규칙이 실제 검증 소프트웨어에서 어떻게 구현될 수 있는지를 보여줍니다.

대표적인 예시는 다음과 같습니다.

- **100_0001**
    - 데이터셋에 피처, 정보 유형, 속성 또는 FC에 정의되지 않은 연관이 포함됨
- **100_0013**
    - CRS가 DirectPosition에 직접 연결됨
- **100_0021**
    - 링이 자기 자신과 교차함
- **100_0024**
    - 링이 폐합(closed)되지 않음
- **100_0030**
    - 고아(Orphaned) 지오메트리 존재
- **100_0048**
    - 좌표 순서가 축(axis) 순서를 따르지 않음
- **100_0053**
    - 필수 레코드·필드·서브필드가 사용되지 않음
- **100_0060**
    - ISO 8211 레코드 순서가 S-100 또는 제품사양 기준을 따르지 않음

이러한 항목들은 Feature Catalogue 일치성, Geometry 무결성, CRS 처리, 좌표 순서, 인코딩 구조, 필수 레코드 존재 여부 등 모든 S-100 제품에 공통 적용되는 기본 검증 요소를 보여줍니다.
