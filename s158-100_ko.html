S-158:100 공통(Generic) 검증 규칙

Validation Checks – Generic Checks

S-158:100은 S-100 기반 데이터 제품을 검증하는 소프트웨어가 반드시 구현해야 하는 공통 검증 항목을 정의한 규칙으로, S-100 Edition 5.2.0 이상을 기반으로 하는 모든 제품표준에 공통 적용됩니다.

표준 개요

S-100 기반 데이터 제품을 검증하는 소프트웨어가 반드시 구현해야 하는 공통(Generic) 검증 항목을 정의한 규칙임. S-100 Edition 5.2.0 이상을 기반으로 작성된 모든 제품표준의 데이터셋 및 교환셋 전체에 적용됨.

Geometry, Attribute, Metadata 등 서로 다른 제품에서 반복적으로 사용되는 개념들에 대해 일관된 검증 품질을 확보하는 역할을 수행함. S-158:1xx(제품별 검증 규칙) 및 S-158:98(상호운용성 검증 규칙)과 함께 사용되어야 하는 공통 기반 규칙임. 전체 S-158 검증 체계 중 공통 규칙 층을 담당함.

적용목적 및 대상

  • 다수의 S-100 제품(S-101, S-102, S-104, S-111 등)이 공통적으로 준수해야 하는 검증 항목을 정의함
  • 서로 다른 제품에서 동일 개념(Geometry, Attribute, Metadata 등)을 사용할 때 일관된 검증 품질을 확보함
  • 데이터셋 및 교환셋 전체를 공통 구조 아래에서 검증함

적용 대상: S-100 Edition 5.2.0 이상 기반 모든 제품표준의 데이터셋 및 교환셋

버전 업데이트 타임라인

  • 2025 / Version 1.0.0 — 피드백 적용 및 더 많은 검사항목에 표준 용어 적용
  • 2024 / Version 0.2.0 — 검사 항목 목록 정비, specification/data 섹션 분리, Part 10b 및 Part 15 검사 항목 추가, 공간 연산자 허용오차 설명 보완
  • 2024 / Version 0.1.0 — 초안 작성

모델 구성

제품 데이터 모델이 아닌, 검증 항목의 구조·검증 순서·공통 검증 범주를 정의하는 규칙 체계임. 각 검증 항목은 S-158에서 정의한 공통 구조를 그대로 따름.

체크 번호 규칙: 100_<4자리번호><선택적 소문자> 형식 (예: 100_0001, 100_0021a)

체크 컬렉션

  • Collection A: 데이터셋·교환셋 검증용 — 실제 검증 소프트웨어가 데이터셋 및 교환셋을 검사할 때 수행해야 하는 항목
  • Collection B: 제품명세·FC·PC·GML Schema 작성 검증용

단일 데이터셋 검증 순서

  • 1a Spatial integrity — 공간 무결성 검사: CRS, Geometry 유효성, 좌표 범위, Tolerance
  • 1b Thematic integrity — 속성 무결성 검사: Attribute 값, datatype, enumeration, multiplicity
  • 1c Association integrity — Feature/Information 관계 무결성 검사: 연관 존재 여부, role, target 일치성
  • 1d Dataset structural conformance — 데이터 구조 검사: Dataset ID, Metadata 구조, Part 10 규격 준수 여부
