S-158 검증 규칙 프레임워크

Validation Checks – Introduction and Structure

S-158은 S-100 기반 데이터 제품의 검증 규칙 체계를 정의하는 프레임워크 표준으로, S-158:100·S-158:1xx·S-158:98 등 파생 규격의 공통 구조와 작성 기준을 제시합니다.

표준 개요

S-100 기반 데이터 제품의 검증 규칙 체계를 정의하는 프레임워크 표준임. 정식 명칭은 Validation Checks – Introduction and Structure이며, Edition 1.0.0은 2025년 2월 발행됨.

개별 데이터 제품을 직접 정의하는 문서가 아닌, 검증 규칙 문서의 구조와 작성 기준을 제시하는 상위 규격임. 데이터 자체보다 검증 규칙의 구조·문법·분류·적용 순서를 표준화하는 데 초점을 둠.

S-158 시리즈 구성

  • S-158: 검증 규칙 프레임워크 (본 문서)
  • S-158:100: 공통(Generic) 검증 규칙
  • S-158:1xx: 제품별 검증 규칙
  • S-158:98: 제품 간 상호운용성 검증 규칙

모든 S-158:1xx 문서는 S-158의 구조와 원칙을 따라야 함.

적용목적 및 대상

목적

  • S-100 기반 데이터셋과 교환셋의 정확성·완전성·무결성을 검증하기 위한 기준을 제공함
  • 제품 내부 오류, 제품 간 충돌, 패키지 구조 오류를 체계적으로 점검할 수 있도록 함
  • 최종 사용자 시스템에서 사용 가능한 수준의 데이터 품질 확보를 지원함

대상

  • S-100 기반 데이터셋(dataset) 검증
  • 교환셋(exchange set) 및 관련 metadata·signature 검증
  • S-128 기반 product catalogue dataset의 정합성 확인
  • 주요 활용 주체: 데이터 생산자·집성자·배포자·응용시스템 및 검증 소프트웨어 개발자

버전 업데이트 타임라인

  • 2025.02 / Version 1.0.0 — 첫 구현 및 시험(implementation and testing)용 Edition 발행
  • 2024.11 / Version 0.2.0 — 검토 의견 반영, S-100 WG9 결정사항 반영, check classification 관련 문구 수정, S-158:1xx 유지관리 체계 개정
  • 2024.08 / Version 0.1.0 — 초안 작성

모델 구성

각 검증 항목은 표 형식의 columnar structure로 정의됨. 필수 구성 요소는 다음과 같음.

  • Check ID: 고유 식별 번호 (1XX_nnnn 형식)
  • Classification: Critical / Error / Warning 구분
  • Check Message: 오류 발생 시 사용자에게 전달되는 안내 문구
  • Check Description: 검사 조건 및 위반 상황 기술
  • Check Solution: 위반 상황에 대한 해결안
  • Reference: 참조하는 규격 및 문서 조항

이력성 항목(Introduced, Modified, Deleted)은 선택적으로 추가할 수 있음.

검증 활동 구분

  • 단일 데이터셋 내부 검사 (a~e)
  • 동일 제품 내 데이터셋 간 검사 (f)
  • 제품 간 상호운용성 검사 (g)
  • 패키지·교환셋·카탈로그 검사 (h~i)

권장 검증 적용 순서

S-158:100 공통 검사 → S-158:1xx 제품별 검사 → S-158:98 단일 데이터셋 상호운용성 검사 → 동일 제품 내 검사 → 제품 간 검사 → 교환셋 검사 → 카탈로그 검사

벡터 제품 검증을 위한 공간 관계 연산자

EQUALS, DISJOINT, TOUCHES, WITHIN, OVERLAPS, CROSSES, INTERSECTS, CONTAINS, COVERED_BY, COINCIDENT
